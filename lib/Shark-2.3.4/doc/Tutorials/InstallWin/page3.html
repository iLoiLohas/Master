Compilation

Choose Build/Batch Build from the menu.

In Visual Studio 6.0 press Rebuild All:

If you are using some later version of Visual Studio you will have to first Select All and then Rebuild:

The compilation starts and produces release and debug object files placed in the folder [SharkRoot]\win_tmp. Depending on your compiler several warning messages appear, dealing with truncated debug symbols (resulting from the usage of the standard template library), exception handling, dll export, and other things. These warnings are clearly compiler specific and can safely be ignored.

After several minutes the linker outputs two static libraries which can be found in

[SharkRoot]\lib\WinNT\Release\Shark.lib
[SharkRoot]\lib\WinNT\Debug\Shark.lib

If you have chosen the workspace Shark_All.dsw the example programs in [SharkRoot]\examples are compiled next. The executables are placed, package wise, into the folders

[SharkRoot]\examples\[package]\WINNT\Release
[SharkRoot]\examples\[package]\WINNT\Debug

If you decided to compile only the libraries for now, it is possible to compile the examples later as needed. Then open the workspace in the folder [SharkRoot]\examples\[package] for the package of interest. For example, [SharkRoot]\examples\LinAlg\LinAlg-Examples.dsw contains all examples for the LinAlg package.
