void MixtureLinearRegression::leaveOneOut ( const Array< double > &  x,
const Array< double > &  y,
Array< double > &  yout 
)

Temporarily removes the vector pair(s) "x"/"y" from the regression model and uses this modified model to approximate the vector(s) "y", when the vector(s) "x" is/are given.

This method can be used to check if some vector pairs are necessary as information for the training of the regression model or not.
After the call of this method, the regression model is reset to the further state, where the pair(s) were included in the model.

Parameters:
xThe x-vector(s) that will be temporarily removed from the model.
yThe corresponding y-vector(s) that will be temporarily removed from the model.
youtThe y-vector(s) for the given vector(s) x, approximated by the modified model.
Returns:
none
Exceptions:
check_exceptionthe type of the exception will be "size mismatch" and indicates that x and y are not one- and 2-dimensional or that one of both is one-dimensional and the other one 2-dimensional or that the number of vectors in x is different to that in y

Given the correlations of the n-dimensional data vector "x" and the m-dimensional data vector "y" and also given their mean values, this function summarizes the data by finding a linear mapping that will approximate the data.

This function is used as subroutine for the class MixtureLinearRegression, where you can directly work with the data vectors and itself and the correlation and mean values are calculated for you.
Please refer to the reference of this class for more detailed information about linear regression.

Parameters:
cxxMatA matrix, that contains the covariances between the single dimensions of the vector . Only the lower triangle matrix must contain values.
cxyMatA matrix, that contains the covariances between the single values of and .
mxVecA-dimensional vector that contains the mean values of vector .
myVecA-dimensional vector that contains the mean values of vector .
amatA matrix, that will contain the transposed transformation matrix , that is used for the linear mapping of the data.
bvecAThe -dimensional vector , that is used for the linear mapping of the data.
dvecA-dimensional temporary vector, that will contain the eigenvalues of matrix cxxMatA in descending order.
Returns:
none
Exceptions:
check_exceptionthe type of the exception will be "size mismatch" and indicates that mxVecA or myVecA are not one-dimensional or that the dimensions of the matrices cxxMatA or cxyMatA don't correspond to the sizes of mxVecA and myVecA (the size of the first dimension of cxxMatA and cxyMatA and the size of the second dimension of cxxMatA must be the same than the size of mxVecA and the size of the second dimension of cxyMatA must be the same than the size of myVecA)

void MixtureLinearRegression::remove ( const Array< double > &  x,
const Array< double > &  y 
)

Removes the data vector pair(s) "x"/"y" from the regression model.

Given the data vector and its corresponding , the internal variables from which the model is calculated are changed, i.e. the information given by and is removed from these variables.
You can also remove several x/y-vector-pairs at once.
The model itself is not changed here, it will changed after calling method update.
This separation is more efficient, e.g. when you are removing several data vectors at different time, because the model itself is then calculated only once after all vectors are removed instead of each time after a single vector pair is removed.

Parameters:
xVector(s) containing the first component of the data.
yVector(s) containing the second component of the data.
Returns:
none
Exceptions:
check_exceptionthe type of the exception will be "size mismatch" and indicates that x and y are not one- and 2-dimensional or that one of both is one-dimensional and the other one 2-dimensional or that the number of vectors in x is different to that in y or that the model was not calculated yet or that it was not calculated for as many vectors as there are vectors in x and y

void MixtureLinearRegression::update ( )

Calculates the current regression model.

Based on internal variables the current regression model is calculated if these internal variables have changed before due to adding/removing vector pair(s). This method is called automatically when you are calling the methods for returning the regression coefficients or or when you are calling the mehod for approximating an for a given (methods A, b, recall), so you can be sure that always the most current model is used.
