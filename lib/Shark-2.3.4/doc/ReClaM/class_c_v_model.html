Detailed Description

Collection of sub-models for cross validation.

The CVModel class is based upon a set of models. The idea is to have one independent model for every partition of the data during the cross validation procedure. The class simply collects these base models and synchronizes its parameters, as long as they are accessed via the CVModel class.
In principle, it is not clear which base model to use for prediction on unseen data. However, it makes sence to use the base models on the whole cross validation data set, that is as well on the subsets used for training, as well as on the subsets for validation.

Constructor & Destructor Documentation

CVModel::CVModel ( Array< Model * > &  models)

Constructor.

This is the default constructor of a cross-validation model. An array containing one base model per fold is provided as a parameter. CVModels constructed this way can be used to make predictions using setBaseModel, model, and modelDerivative.
Parameters:
modelsArray of identical base models. There must be as many models in the array as there are cross validation partitions.

Constructor.

This simplified constructor makes the CVModel use the same underlying base model for each fold. The model can be evaluated via CVError::error and CVError::errorDerivative, but the results of calls to setBaseModel, model, and modelDerivative are undefined.
Parameters:
