Detailed Description

Meta Model for SVM training.

Author:
T. Glasmachers
Date:
2006
The C-SVM is a training scheme for Support Vector Machines. It defines a positive constant C controlling the solution complexity.
In the 1-norm SVM formulation, this constant bounds the parameters of the SVM from above, limiting their contribution to the solution. For the 2-norm SVM formulation, the are not bounded from above. Instead, the kernel matrix is reglarized by adding to the diagonal entries.
The C_SVM class is a meta model which is based on an SVM model. Its parameter vector is composed of the SVM hyper- parameters, i.e., the complexity parameter C and the parameters of a KernelFunction object. As a meta model it does not make sense to call the model or modelDerivative members of this class. However, the modelDerivative function has been implemented to compute the derivative of the underlying SVM model w.r.t. the hyperparameters stored in the C_SVM model. See PrepareDerivative and modelDerivative for further reference.
For unbalanced data, it has proven fertile to consider class specific complexity parameters and instead of a single constant C. These parameters are usually coupled by the equation , where ${}$ are the class magnitudes. Therefore, the C_SVM model introduces only the parameter as a model parameter, whereas can be computed from this parameter. As only the parameter is subject to optimization, all derivatives have to be computed w.r.t , even if is used for the computation.
The parameter can be stored in a way allowing for unconstrained optimization, that is, the exponential function is used to compute the value from the model parameter. As a side effect, this parameterization is more natural and therefore probably better suited for optimization.
Examples:

CrossValidation.cpp, KernelOptimization.cpp, SvmApproximationExample.cpp, SVMclassification-gnuplot.cpp, and SVMclassification.cpp.
