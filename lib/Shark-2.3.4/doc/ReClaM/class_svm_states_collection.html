Detailed Description

Utility class for storing snapshots of SVM states.

When training an SVM (solving its QpProgram), it can be interesting to analyze the intermittant solver states and thereby its learning curve. An SvmStatesCollection instance provides storage to which the relevant parameters can be pushed during training. The parameter copy procedure is non-sparse, so that copy time is constant during model buildup. All stored model snapshots can be sparsified after the optimization run. The storeXXX methods allow triggering calculations of more resource-intensive measures after training and sparsification, while the XXXdirectly methods are for quick direct storage of cheaply obtainable measures from within the solver.
