Vector and Matrix Classes

The LinAlg module builds on Arrays, in particular on the class Array<double>, to represent both Vectors and Matrices. Alternatively, LinAlg provides a more object oriented interface to these concept in terms of two specialized classes:

  • class Vector : public Array<double>     [reference]
  • class Matrix : public Array<double>     [reference]
This tutorial teaches how these classes are used and introduces their capabilities beyond simple arrays.

The classes are accessible after including the header 

#include <LinAlg/VecMat.h>

Array Access

As a rule of thumb you may think of Vectors as one-dimensional instances of Array<double> while, of course, a Matrix object is two-dimensional. All access operations from the superclass Array<double> are available, but some functionalities have been overloaded in order to reflect the special needs of vectors and matrices.

Size information

In contrast to simple Arrays, the dimensions of vectors and matrices are fixed. Therefore it is impossible to construct these objects with a default constructur. Instead, the programmer is forced to provide the necessary size information already at construction time. Of course it is possible to resize vectors and matrices later, but the number of dimensions is always kept correct.
