Defining the Learning Problem

This tutorial serves as a primer for using the Shark implementation of feed-forward multi-layer perceptron neural networks. The whole functionality can be discovered in the ReClaM documentation. It is recommended to read the mini-tutorials on how to compile a program using Shark and on Arrays first.

We start with solving the infamous XOR-problem using a standard feed-forward multi-layer perceptron network.

First we define the problem by generating the training data. We consider to binary inputs that are to be mapped to one if they have the same value and to zero otherwise. Input patterns and corresponding target patterns are stored in separate Arrays. 

#include <Array/Array.h>

int main()
{
  // just index variables
  unsigned i, j, k, t;
  Array<double> trainInput(4, 2);
  Array<double> trainTarget(4, 1);

  // construct xor problem
  for(k=0, i=0; i<2; i++) {
    for(j=0; j<2; j++) {
      trainInput(k, 0) = i;
      trainInput(k, 1) = j;
      trainTarget(k, 0) = (i+j) % 2;
      k++;
    }
  }

  return EXIT_SUCCESS;
}

There are four different input patterns. For example, trainInput(3, 0) and trainInput(3, 1) correspond to the values of the first and second input variable of the input pattern (1,1). The corresponding target trainTarget(3, 0) is 0.
