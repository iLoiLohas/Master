Detailed Description

template<class T>
class RCPtr< T >

Template class for Reference Counted Pointers.

When we create objects like FuzzySets for instance it is hard to decide who should be owner of the object and responsible for freeing the memory again, since the FuzzySet can be used at many different locations, e.g. as part of a ComposedFS again. Problems like these are solved by the concept of Reference Counted Pointers (see: Scott Meyers, More Effective C++). RCPtr objects keep track of the number of references to the object and frees it when the last reference is deleted.

But unfortunately there are disadvantages as well. For instance, this concept prevents us from using the usual polymorphism:

RCPtr<FuzzySet> fs( new BellFS( 1, 0 ) );

So, in this case 

fs->defuzzify()

would mean the method from FuzzySet, not the one from BellFS.

Note: T must inherit from RCObject to be handled by RCPtr.
